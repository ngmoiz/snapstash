# Contributing

## Branch naming convention

`<type>/<brique>-<short-description-in-kebab-case>`

#### Examples:
```
docs/b0.1-gitignore-and-readme
feat/b1.4-health-endpoint
chore/b0.1-project-setup
```

#### Types:

| Type     | How to use it |
| -------- | ------- |
| feat     | a new feature |
| fix      | fix a bug |
| docs     | only documentation (README, ROADMAP ...) |
| chore    | maintenance tasks |
| ci       | GitHub Actions pipelines |
| refactor | refactor without major change |

#### Rules

- all lowercase, separate words with hyphens (kebab-case)
- no space, no uppercase, no accent
- not too long, 50 characters max
